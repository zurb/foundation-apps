Foundation for Apps AlphaGrid Docs

Welcome!

These are the docs for the alpha version of the Foundation for Apps grid. This is a flexbox-based grid system that supports horizontal and vertical layout and is intended to provide a more robust, app-centric way of handling layout.

If you have feedback on the grid please report issues to the Foundation for Apps repo. If there is an existing issue covering your topic please use that so we can keep the issue list clean. Pull requests are welcome from the core team and invited contributors, but be explicit (not that kind) so we know what's what. Thanks!

The Basics: Group and Block

The most basic elements in the Foundation for Apps grid are the Group and Block.

Groups

Groups are layout objects that contain either more groups, or blocks. The are roughly analogous to rows in the Foundation 5 grid.

Nesting Groups

Groups can contain other groups to further split the layout, much like nesting columns inside rows in previous Foundation versions.

Blocks

Blocks are layout objects that contain content only. They are not a layout object, but should contain other elements and components. These are roughly analagous to columns from Foundation 5.

Sizing Elements

Sizing elements in Foundation for Apps is done similarly to Foundation for sites, with a twelve column grid you can apply to objects.

In Foundation for Apps, however, you can apply sizes to both groups and blocks (so to rows and columns) and nest them as far down as you need.

Group Sizes

Putting sizes on a group is how we nest the grid down — you always want to use groups when you are going to put either deeper nested groups or blocks within them.

Block Sizes

Block sizes work like you would expect, however you are not restricted any longer to twelve columns in a row — you can have as many blocks in a group as you'd like and they will flow down as their sizes fit in twelve units.

Nested Sizes

When you nest groups to contain blocks, you can assign sizes to every element in that chain. The example here shows a group which has been split into two groups and in turn split into component blocks for content.

Stack / Unstack

By default, blocks will line up left to right in a group not unlike the columns inside rows of previous Foundation versions. However you now have the option to have them stack themselves vertically while still being sized along that axis.

Stacking Groups and Blocks

By adding a class of .[size]-stack to a group, the groups or blocks inside it will be aligned along the vertical or Y axis, and sizes you specify will affect the vertical height of the object rather than width.

Note: You can use `.stack` as an alias for `.small-stack`.

Unstacking Groups and Blocks

If for example you stack your group on small devices, but want a horizontal layout on larger devices, you can unstack them (return them to a horizontal flow) by applying a .[size]-unstack class to the group.

Note: You can use `.unstack` as an alias for `.small-unstack`.

Frames

A frame is a layout object that lives above all others, and is 100% of the width and height of the screen you're on. This allows you to carve the screen up into areas that are independently scrollable.

Creating a Frame

You can create a frame by attaching the class to the body of the page, or a div (or other block level element) inside the body. You should not put frames inside groups or blocks further down in your nesting as they are sized in viewport units.
